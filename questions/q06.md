Question 6 — OneHotEncoder usage and output

Step‑by‑step solution
1) Create the encoder and prepare the input as a 2‑D array because `OneHotEncoder` expects `(n_samples, n_features)`.
2) Fit on the single categorical feature, then transform to a dense one‑hot array.
3) By default, categories are discovered and ordered lexicographically, so the column order becomes `[blue, green, red]` for the provided values.

Code
```python
import numpy as np
from sklearn.preprocessing import OneHotEncoder

encoder = OneHotEncoder(sparse=False, handle_unknown='ignore')
X = np.array(['red', 'blue', 'green', 'blue']).reshape(-1, 1)
result = encoder.fit_transform(X)

print(encoder.categories_)  # [array(['blue', 'green', 'red'], dtype='<U5')]
print(result)
# array([
#   [0., 0., 1.],  # red
#   [1., 0., 0.],  # blue
#   [0., 1., 0.],  # green
#   [1., 0., 0.],  # blue
# ])
```
